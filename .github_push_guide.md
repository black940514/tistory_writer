# GitHub Push 가이드

## 1. Git 초기화 및 GitHub 연결

```bash
cd /Users/kimtaeyoun/Personal/Dev/Tistory_writer

# Git 초기화 (처음인 경우)
git init

# .gitignore 확인 (config.yaml, data/ 등이 제외되는지 확인)
cat .gitignore

# GitHub에서 새 저장소 생성 후 원격 저장소 추가
git remote add origin <your-github-repo-url>
# 예: git remote add origin https://github.com/yourusername/tistory-auto-poster.git
```

## 2. 파일 추가 및 커밋

```bash
# 모든 파일 추가 (.gitignore에 의해 config.yaml, data/ 등은 자동 제외됨)
git add .

# 커밋
git commit -m "Initial commit: Tistory auto poster with scheduler"

# GitHub에 푸시
git push -u origin main
```

## 3. 확인사항

다음 파일/디렉토리는 GitHub에 업로드되지 않습니다 (의도된 것):
- `config.yaml` (민감한 정보 포함)
- `prompts.yaml` (커스텀 프롬프트)
- `data/` (논문 리스트, 상태 파일 등)

이들은 리눅스 서버에서 직접 생성해야 합니다.

## 4. 리눅스 서버에서 배포

GitHub에 push한 후 리눅스 서버에서:

```bash
# 클론
git clone <your-github-repo-url>
cd Tistory_writer

# 설정 파일 생성
cp config.yaml.example config.yaml

# 설정 파일 편집 (스케줄러 활성화 필수!)
nano config.yaml
# schedule.enabled: true 로 변경

# Docker 실행
docker compose up -d --build

# 로그 확인
docker compose logs -f
```
